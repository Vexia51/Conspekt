SSH (Secure Shell) - это сетевой протокол для безопасного удаленного управления компьютером через шифрованный канал на порту 22. Он защищает пароли и команды от перехвата в сети.

При подключении SSH использует асимметричное шифрование (пару из публичного и приватного ключей) для безопасного обмена данными и проверки подлинности. После установки соединения включается симметричное шифрование, которое кодирует весь основной трафик одним общим ключом, а хеширование HMAC гарантирует, что данные не были изменены в пути.

Авторизоваться в системе можно по обычному паролю или по ключам, что гораздо безопаснее. При авторизации по ключам ваш публичный ключ копируется на сервер в файл ~/.ssh/authorized_keys, а приватный ключ остается у вас и подтверждает вашу личность без отправки пароля в сеть.

Для подключения используется команда ssh user@host. Создать новую пару ключей можно через `ssh-keygen -t ed25519`, а отправить публичный ключ на удаленную машину - с помощью ssh-copy-id user@host. Пересылка файлов выполняется через команду scp file.txt user@host:/path/.

Для защиты сервера в файле конфигурации /etc/ssh/sshd_config обычно меняют стандартный порт на любой случайный, запрещают удаленный вход для суперпользователя строкой PermitRootLogin no и полностью отключают вход по паролям через PasswordAuthentication no, разрешая подключение только по ключам.
