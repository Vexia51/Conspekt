#RFB #TightVNC
RFB (Remote Framebuffer) - платформонезависимый протокол для удаленного управления графическим интерфейсом. Сервер передает клиенту изменения пикселей экрана, а клиент отправляет обратно сигналы клавиатуры и мыши. Протокол работает на прикладном уровне (модель OSI) и использует по умолчанию порт TCP 5900. Для снижения трафика применяются различные алгоритмы сжатия (ZRLE, Tight).

Для установки соединения на удаленном ПК должен быть запущен TightVNC Server и задан пароль доступа. На локальном компьютере открывается утилита TightVNC Viewer, где в поле Remote Host вводится IP-адрес или доменное имя целевого компьютера (при необходимости через двоеточие указывается порт). После инициализации соединения запрашивается пароль, установленный на сервере, после ввода которого открывается рабочий стол удаленной системы и осуществляется полный перехват управления клавиатурой и мышью. В процессе работы доступна панель инструментов, позволяющая отправлять команду Ctrl+Alt+Del, переходить в полноэкранный режим и снижать качество цветопередачи для ускорения отклика при медленном интернете.
